blockr.bi renderers — the model

The converged concept. Tables, charts and tiles are renderers over a rectangular data frame; all shaping (summarize, pivot_wider, correlate) happens upstream. Switching how you see the data is switching the renderer, not rebuilding the pipeline.

Final concept · 2026-06-13 · clean rebuild · companions: exploration mockup · what-others-do research · dev/table-and-chart-architecture.md

A renderer maps columns → roles and draws. It never aggregates, reshapes, or computes.

One tidy frame can be drawn as a table, a chart, or a tile — interchangeable views. Pick the renderer by what you want to show; the data is the same.

01Shaping upstream, drawing downstream

The upstream verbs are the product. The renderer is the last node and the only one that touches pixels.

03The tile — slots, not a fixed schema

A tile shows a main value surrounded by slots you map to columns. Crucially, there is no delta/target/max element — those were arbitrary. There's a generic secondary value (any reference column: last-month mean, a target, prior year, a benchmark) and a display style that says how to draw it.

Revenue · Q3 1

$1.24M 2
▲ 12.7% 3

84% of $1.10M target 3

strongest quarter since 2024 5

1 overline — supertext; free text or a column
period, category, source — your “fill with whatever”
2 value — the main number, with format (%, $, compact) + unit
3 secondary ×N — any reference column, each with a display style:
plainΔ% vs valuefill / progresspill
“delta”, “target”, “max” all collapse into here — they were just secondary + a style
4 color — tint or sign-coloring, auto from a column’s value/sign
5 caption — subtext; free text or a column
Why no fixed delta: a ▲% vs last month is just one comparison — last-year, a target, a peer average are equally interesting. Forcing them through a “delta” field is the rigidity that makes fixed schemas bolt arrows onto prefix. The number stays generic (any column); only the rendering is typed (the display style gives you the auto % and the green/red).

Two layouts, identical slots. Cards stack the slots; the table lines them up. Nothing exists in one layout that can’t in the other.

04The table — one renderer for every table

Table 1, a regression table, a crosstab, a heatmap — the same display: rows × an optional spread dimension × cells. Structure rides in hidden dotted columns (.section, .indent, .fmt); numbers stay numeric and compose at draw time.

The vocabulary: shape upstream (summarize / correlate / pivot_wider) → render with one of a few deep, composable renderers (table, chart, tile). The tile is slots (overline · value · secondary×N+style · color · caption), two layouts (cards · table). One frame, many views.
