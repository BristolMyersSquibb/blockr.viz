One self-contained file — inline CSS, no framework. JS drives only the count-up, bar grow-in, and the light/dark toggle (which satisfies variant F). Out of scope: sparklines / trend lines (a separate chart component owns those) and any data fetching — this stays presentational. Tokens build on blockr.dock greyscale and the #2563eb primary; dark mode is composed from surface-elevation tokens, not inverted colors.
